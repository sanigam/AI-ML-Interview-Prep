# Computer Vision: Object Detection and Segmentation

## Interview Anchor
- **Object Detection Fundamentals:** Combining classification (what) and localization (where) in a unified framework; key challenges in handling variable-sized objects and backgrounds
- **Detector Architectures:** Two-stage detectors (propose regions → classify) versus one-stage detectors (classify and locate simultaneously); accuracy vs latency tradeoffs
- **Segmentation Tasks:** Semantic (classify each pixel), instance (classify and separate objects), and panoptic (combine both); architectural designs and applications

## Key Concepts Overview
Object detection and segmentation represent some of the most practical computer vision tasks, deployed in autonomous vehicles, medical imaging, and surveillance. Interviewers assess whether you understand the evolution from R-CNN to modern efficient detectors (YOLO, SSD), why one-stage detectors are faster but two-stage detectors more accurate, and how to design end-to-end systems handling variable object sizes and class imbalance. Strong candidates explain the distinction between detection (bounding boxes), semantic segmentation (pixel-level classification), and instance segmentation (separate objects), understand key architectural innovations like Feature Pyramid Networks and anchor-free detection, and discuss practical optimizations for real-time deployment.

### Q1: Formulate object detection as a computer vision problem; what are the key challenges compared to image classification?

**A:** Object detection combines two tasks: classification (what object is in the image?) and localization (where is it?). Unlike classification (one prediction per image), detection requires multiple predictions—each object needs a category label and bounding box. Key challenges: (1) Variable number of objects—a detector must produce a variable-length output (0 to N objects), unlike classification's fixed output, (2) Variable scales—objects range from tiny to filling the image; detectors must handle this without explicit multiscale processing, (3) Class imbalance—background dominates foreground (>99% of region proposals are background in typical images), (4) Localization precision—pixel-level accuracy is harder than class prediction, requiring precise bounding box regression, (5) Speed-accuracy tradeoff—detection is inherently slower than classification (needs to examine many regions), and real-time detection is challenging. Example: In autonomous driving, a detector must find pedestrians (1% of pixels), cars, and cyclists at variable scales (distance), predict tight bounding boxes (for path planning), and run in <100ms. Contrast with ImageNet classification where the task is simply "is this a dog?"—no localization needed.

### Q2: Explain anchor-based detection and compare it with anchor-free approaches.

**A:** Anchor-based detection predicts offsets from predefined anchor boxes (default boxes of various aspect ratios and scales tiled across the image). For each location and anchor, the detector predicts: (1) confidence that an object exists in this anchor, (2) class probabilities if an object exists, (3) bounding box offsets (dx, dy, dw, dh) to adjust the anchor box. Advantages: simple to implement, leverages explicit geometric priors, widely used (YOLO v3, SSD, Faster R-CNN). Disadvantages: sensitive to anchor box design (wrong anchor sizes hurt performance), generates many default boxes (computational cost), and different anchor designs work better for different datasets. Anchor-free approaches (YOLO v1, CenterNet, FCOS) directly predict object centers and their size, without predefined anchors. Instead of anchors at each location, the detector predicts if a point is an object center, and if so, the object's bounding box. Advantages: no hyperparameter tuning for anchor sizes, cleaner training (each object assigned to one location), better for objects with irregular aspect ratios. Disadvantages: more difficult optimization (learning to predict center location from scratch), historically lower accuracy (though recent methods improved this). Example: Faster R-CNN uses anchors of scales [128², 256², 512²] and ratios [1:1, 2:1, 1:2] at each spatial location. FCOS (anchor-free) predicts at each spatial location whether that location is an object center, and if so, the distance to the object's boundaries. Modern trend: anchor-free methods are gaining popularity (YOLOv6+) due to simpler design and competitive accuracy.

### Q3: Explain the two-stage detection pipeline: proposal generation and classification (R-CNN family).

**A:** Two-stage detectors separate detection into: (1) Region Proposal Network (RPN): generate candidate regions (thousands) that might contain objects, using sliding window or selective search, (2) Classification and refinement: crop each proposal, classify its content, and refine the bounding box. R-CNN (original): used selective search to generate ~2000 region proposals, extracted CNN features for each proposal (slow—forward pass per region!), classified with SVM, refined bounding boxes with regression. Fast R-CNN improved efficiency by extracting features once from the whole image, then pooling features corresponding to each region proposal (RoI pooling), classifying and refining in parallel. Faster R-CNN made region proposal generation learned: trained a Region Proposal Network (RPN) as part of the detector that predicts which anchors likely contain objects. Advantages of two-stage: high accuracy (can afford expensive operations per proposal since only thousands, not millions, are evaluated), interpretable (regions clearly identified before classification). Disadvantages: slower than one-stage (two forward passes), requires careful tuning of region count (too many → slow, too few → miss objects). Example: Faster R-CNN on a 1000x1000 image: (1) Extract CNN features (e.g., ResNet-50), (2) RPN generates ~2000 region proposals across scales, (3) Apply RoI pooling to extract 7x7 features from each proposal, (4) Classify each region (256 dimensions fully connected) and refine bounding box, (5) Apply NMS to remove duplicates. Modern two-stage detectors (Cascade R-CNN, Efficient Det) build on this foundation.

### Q4: Describe one-stage detectors: YOLO evolution from v1 to v8, and SSD.

**A:** One-stage detectors predict class and bounding boxes directly without region proposals, trading accuracy for speed. YOLO v1 (2015) divides the image into SxS grid, predicts B bounding boxes per grid cell (with confidence scores) and C class probabilities. The prediction is a single pass through the network, enabling real-time speed (~45 FPS). Weakness: YOLO v1 struggles with small objects and nearby objects in the same grid cell. YOLO v2/v3 improved: used anchor boxes instead of explicit grid predictions, added multi-scale predictions (detecting small objects on high-resolution feature maps, large objects on low-resolution), used batch normalization and better backbone (Darknet-53). YOLO v4 introduced data augmentation (Mosaic, MixUp), GIoU loss for better localization, and cross-stage partial connections. YOLO v5-v8 focused on efficiency and practical deployment: smaller model variants (nano, small, medium, large), inference optimization, export to multiple formats. SSD (Single Shot MultiBox Detector) uses feature maps at multiple scales: predicts on 38x38, 19x19, 10x10, 5x5 feature maps, enabling detection across scales. Like YOLO, it's a one-stage detector predicting class and bounding boxes in one pass. Accuracy: one-stage detectors are typically ~5-10% less accurate than two-stage, but 5-10x faster. Example: YOLO v8m on 640x640 images: ~78 FPS on GPU, mAP around 50 (acceptable for many applications). Trade-off: use two-stage for high-accuracy applications (autonomous driving), one-stage for real-time constraints (mobile, edge devices).

### Q5: What is focal loss and why is it important for one-stage detectors?

**A:** Focal loss addresses class imbalance in one-stage detectors. In a typical image, >99% of predicted regions are background, <1% are foreground objects. Standard cross-entropy loss gives equal weight to easy negatives (background predictions) and hard positives (object predictions), so the model is dominated by easy negatives and ignores learning from positives. Focal loss down-weights easy examples: FL(p_t) = -alpha_t * (1 - p_t)^gamma * log(p_t). The (1 - p_t)^gamma term is a focusing parameter that reduces loss for easy examples (high confidence predictions) and focuses on hard examples (low confidence predictions). Hyperparameter gamma typically 2 controls how much to down-weight easy examples. Example: For a background prediction with p_t=0.99, standard cross-entropy loss is -log(0.99) ≈ 0.01. Focal loss with gamma=2: -(0.01)^2 * log(0.99) ≈ 0.0001—reduced by 100x. For hard positive (confidence 0.5), focal loss gives similar loss to cross-entropy, so learning focuses on hard cases. Result: RetinaNet with focal loss achieves two-stage detector accuracy (average mAP ~40) while maintaining one-stage speed. Focal loss is now standard in one-stage detectors (YOLO v3+, SSD, FCOS). Without focal loss, one-stage detectors severely underperform because background examples dominate training.

### Q6: Explain Non-Maximum Suppression (NMS) and its variants (Soft-NMS, DIoU-NMS).

**A:** Detectors output many overlapping bounding boxes for the same object. NMS removes duplicates by keeping high-confidence predictions and suppressing nearby lower-confidence ones. Standard NMS: (1) Sort predictions by confidence, (2) Keep the highest-confidence prediction, (3) Remove all predictions with IoU > threshold (default 0.5) with the kept prediction, (4) Repeat until done. NMS removes false positives (duplicate detections of the same object) but can fail when objects overlap (suppresses valid detections of nearby objects). Soft-NMS improves this by decaying confidence scores of overlapping boxes instead of removing them: for each suppressed box, reduce its confidence by a factor based on IoU (rather than hard threshold). This keeps overlapping detections with reduced confidence, preserving objects that overlap slightly. DIoU-NMS uses a different metric: instead of IoU (intersection over union), uses DIoU (distance IoU) that also considers distance between box centers. This is better at handling aligned objects (e.g., cars in a traffic jam) where IoU is high but they're different objects. Example: NMS on street scene: one car detected at [100, 100, 150, 150] with confidence 0.9, and nearby prediction at [105, 105, 155, 155] with confidence 0.7. Standard NMS removes the second box because IoU > 0.5. Soft-NMS reduces its confidence to 0.7 * (1 - IoU) ≈ 0.35, so it's still output. DIoU-NMS would check if boxes are actually different objects based on center distance. NMS is post-processing, applied after model prediction, and significantly affects final mAP—using appropriate NMS variant for your domain is important.

### Q7: Define IoU and explain variants: GIoU, DIoU, CIoU.

**A:** Intersection over Union (IoU) measures overlap between predicted and ground-truth bounding boxes: IoU = intersection_area / union_area. Values range [0,1]; IoU=1 means perfect overlap, IoU=0 means no overlap. Standard metric for evaluation: a prediction is correct if IoU > 0.5 (COCO standard is 0.75). Limitation: IoU doesn't punish predictions that don't overlap—two boxes with zero overlap both have IoU=0, so gradient is zero, preventing the model from learning to move the box toward the target. Generalized IoU (GIoU) = IoU - (enclosing_area - union_area) / enclosing_area. This ensures non-overlapping boxes have GIoU < 0, providing gradient signal to move the box toward the target. Distance IoU (DIoU) = IoU - (distance_centers^2 / diagonal_enclosing^2). Penalizes distance between box centers, encouraging centered predictions even when overlap is partial. CIoU (Complete IoU) = DIoU - v^2/1-IoU where v measures aspect ratio difference. Combines overlap, center distance, and aspect ratio penalty. Example: Predicting box [10, 10, 50, 50] for ground truth [100, 100, 140, 140], IoU = 0 (no signal), GIoU ≈ -0.7 (pulls box toward target), DIoU includes distance penalty. In practice: use GIoU or CIoU loss for bounding box regression instead of L1/L2 loss; this improves accuracy by ~1-2 mAP. Most modern detectors (YOLO v4+, EfficientDet) use CIoU loss.

### Q8: Explain Feature Pyramid Networks (FPN) and why they're essential for multi-scale detection.

**A:** Feature Pyramid Networks address the challenge of detecting objects at multiple scales. Backbone networks (ResNet) produce features at different resolutions: early layers have high resolution (16x16 feature map for 256x256 image) but low semantic information, deep layers have low resolution but high semantic content. FPN builds a pyramid: (1) Extract features from multiple backbone layers (C2, C3, C4, C5 at different scales), (2) Create a top-down pathway: start from deepest features (highest semantics), upsample and merge with shallower features via lateral connections (1x1 convolutions), (3) Predict detections at each level of the pyramid. This way, predictions are made at multiple scales with high semantic content at each scale—small objects are detected on high-resolution features, large objects on low-resolution features. Example: FPN on ResNet-50: C2 (stride 4, high resolution), C3 (stride 8), C4 (stride 16), C5 (stride 32, low resolution). Top-down path: upsample C5 (32x32 feature map) to match C4's scale (16x16), concatenate, repeat. Result: P2-P5 feature pyramids, each with 256 channels and high semantic content. Detect small objects on P2 (stride 4, 64x64 for 256x256 image—fine detail), large objects on P5 (stride 32, 8x8—semantics). FPN improved mAP by ~3-5% across detectors because every scale gets sufficient semantic information. FPN is standard in modern detectors (Faster R-CNN, YOLO v3+, EfficientDet).

### Q9: Distinguish semantic segmentation, instance segmentation, and panoptic segmentation.

**A:** Semantic segmentation assigns a class label to each pixel; outputs are class predictions per pixel. Approach: fully convolutional network (FCN) takes an image, applies convolutions and pooling (losing spatial resolution), then upsamples back to original resolution with skip connections. U-Net (encoder-decoder with skip connections) is popular for medical imaging. DeepLab uses dilated convolutions and atrous spatial pyramid pooling (ASPP) to capture multi-scale context without losing resolution. Limitation: cannot distinguish different instances of the same class (two dogs appear as one "dog" region). Instance segmentation predicts per-instance masks—outputs are object boundaries for each distinct object. Mask R-CNN extends Faster R-CNN: for each region proposal, predict a mask (binary segmentation within bounding box) in addition to class and box. This correctly separates two dogs as two separate instances. Panoptic segmentation combines both: predict class labels and instance boundaries, separately for "stuff" (amorphous regions like sky, wall) and "things" (countable objects like people, cars). Example: Image with two dogs and sky. Semantic segmentation: each pixel is labeled "dog" or "sky"—can't tell the two dogs apart. Instance segmentation: outputs two "dog" masks and one "sky" mask. Panoptic segmentation: outputs class "dog" with instance 1 and 2, class "sky" with instance 1 (stuff doesn't have instances). Panoptic segmentation is more complex but provides complete scene understanding. Task selection depends on application: semantic for simple classification (land use mapping), instance for object counting/tracking, panoptic for autonomous driving (need to identify and separate all objects and scene context).

### Q10: Explain semantic segmentation architectures: FCN, U-Net, and DeepLab.

**A:** Fully Convolutional Networks (FCN) pioneered end-to-end dense prediction. Key idea: replace fully connected layers with convolutions, enabling variable input sizes and dense outputs. Architecture: downsampling path (convolutions, pooling) reduces spatial resolution while increasing channels, then upsampling path (transposed convolutions, unpooling) restores resolution. Skip connections (feature concatenation from downsampling to upsampling) preserve fine details. Limitation: simple bilinear upsampling introduces artifacts. U-Net improves FCN with symmetric encoder-decoder: downsampling extracts context, upsampling restores details with skip connections at each level. U-Net is particularly effective for medical imaging (limited data) because skip connections enable learning of fine details even with small training sets. DeepLab uses dilated (atrous) convolutions to increase receptive field without losing resolution: normal 3x3 convolution sees 9 pixels, but dilated convolution with rate=2 sees 25 pixels spaced apart. ASPP (Atrous Spatial Pyramid Pooling) applies multiple dilated convolutions with different rates in parallel, capturing multi-scale context. Example: U-Net for medical imaging: 64→128→256→512 channels downsampling, then 512→256→128→64 upsampling, skip connections at each level. Input image 572x572, output segmentation mask same size. DeepLab for street scenes: backbone extracts 1/16 resolution features, ASPP applies 1x1, 3x3 (rate=6), 3x3 (rate=12), 3x3 (rate=18) dilated convolutions in parallel, concatenate and predict classes. DeepLab achieves real-time speed with good mAP (~82 on Cityscapes).

### Q11: Explain Mask R-CNN and how it extends Faster R-CNN for instance segmentation.

**A:** Mask R-CNN extends Faster R-CNN by adding a branch that predicts a mask (binary segmentation) for each region proposal. Architecture: (1) Backbone extracts features (ResNet-FPN), (2) RPN generates region proposals, (3) For each proposal, apply RoI pooling (or RoI Align, a differentiable improvement), (4) Parallel branches: classify region (softmax), refine bounding box (regression), predict mask (sigmoid binary classification per pixel). Mask branch: inside each region, predict a mask the same size as the region (e.g., 28x28), indicating which pixels belong to the object. Key innovation: RoI Align instead of RoI pooling—regular RoI pooling quantizes coordinates to grid points (losing precision), RoI Align uses bilinear interpolation to align features to region boundaries precisely, improving mask accuracy. Loss function: combine classification loss, bounding box loss, and mask loss (binary cross-entropy per pixel). Example: Mask R-CNN on COCO: input image 1024x1024, backbone extracts 1024/32=32x32 feature map (stride 32). RPN generates ~1000 proposals. For each proposal (say 128x128 pixels), RoI Align crops and resizes to 14x14 features, mask head upsamples to 28x28 and predicts mask. Output: bounding box, class, confidence, and instance mask. Advantages: straightforward extension to R-CNN family, high accuracy (mAP ~40 on COCO), produces pixel-accurate boundaries unlike bounding box. Disadvantages: slower than two-stage bounding box detection (additional mask prediction per region), but still feasible for many applications.

### Q12: Explain real-time object detection optimization: model efficiency and inference speed.

**A:** Real-time detection requires <30ms per image (for 30 FPS) or <100ms (for 10 FPS, acceptable for some applications). Key bottlenecks: model size (large models are slow), inference hardware (GPU vs CPU), and batch size (batching amortizes computation). Optimization strategies: (1) Model architecture: use efficient backbones (MobileNet, ShuffleNet, SqueezeNet) instead of ResNet-50, reducing parameters by 10-50x with acceptable accuracy drop. (2) Model pruning: remove unimportant weights, reducing inference time with minimal accuracy loss. (3) Quantization: convert float32 to int8 (4x memory reduction, 2-4x speedup, typically <1% accuracy loss). (4) Knowledge distillation: train smaller student model to mimic larger teacher model, enabling the student to be faster while preserving accuracy. (5) Inference optimization: use optimized frameworks (ONNX, TensorRT) that fuse operations and auto-tune for hardware. (6) Batch processing: process multiple images simultaneously to improve GPU utilization. Example: YOLO v8n (nano) is 50MB, runs at 640x640 in ~2ms (500 FPS GPU, 30ms CPU)—suitable for mobile. YOLO v8x (large) is 130MB, runs at 640x640 in ~20ms (50 FPS GPU)—suitable for high-accuracy applications. For embedded systems (edge cameras), use quantized MobileNet backbone: achieves 30 FPS at <100ms with <30MB model. Real-time constraint usually forces accuracy-speed tradeoff: you cannot have state-of-the-art mAP at 30 FPS, must choose.

### Q13: Describe Vision Transformers for detection: DETR and DINO.

**A:** Vision Transformers (ViT) replace CNNs with transformer self-attention for detection. DETR (Detection Transformer, 2020) reformulates detection as a sequence prediction problem: embed image patches with a CNN backbone, pass through transformer encoder-decoder, output N object predictions (each with class and bounding box). Unlike CNNs that predict grids of boxes, DETR outputs N objects directly—no need for NMS (non-maximum suppression) post-processing, since decoder learns to output non-overlapping predictions. Architecture: CNN backbone (ResNet) extracts image features (e.g., C5 at stride 32), flatten spatial dimensions, add positional encodings, pass through transformer encoder-decoder. Decoder uses object queries (N learned embeddings) that interact via self-attention, updated iteratively by the decoder layers. Each query corresponds to one object, predicting its class and bounding box. Advantage: simpler architecture (no anchors, no NMS, end-to-end differentiable), interpretable (attention maps show which image regions each query attends to). Disadvantage: slower than CNN detectors initially (requires all N queries to interact, expensive), and initial DETR had lower accuracy on small objects. DINO (DETR with Improved deNoising, 2022) improved DETR: added denoising training (masking some queries to denoise corrupted predictions), improved architecture, and uses more efficient attention. Result: DINO achieves higher accuracy than CNN detectors while remaining practical. Modern trend: vision transformers are closing the gap with CNNs, especially for large-scale detection. ViT backbones (ViT-B, ViT-L) now competitive with ResNet on both accuracy and speed when properly optimized.

### Q14: Explain data augmentation for object detection: Mosaic, MixUp, CutOut, and why they matter.

**A:** Standard image augmentation (random crop, horizontal flip, brightness change) is insufficient for detection because objects can be lost during augmentation. Detection-specific augmentations ensure objects remain visible and properly augmented. Mosaic combines 4 random images into one: take 4 images, resize to half-size, arrange in a 2x2 grid. This increases batch diversity without requiring larger batch size, and forces the model to detect objects at different scales/positions within the same image. Effective especially with small objects—mosaic ensures some objects appear at different positions. MixUp mixes two images (pixel-level interpolation): output = alpha * image1 + (1-alpha) * image2, alpha ~ Uniform[0,1]. Also mix labels: if image1 has "dog" and image2 has "cat," both labels are present. Forces the model to learn from partially mixed objects. CutOut removes rectangular regions from images (sets pixels to zero or gray). Encourages the model to detect objects partially obscured, and prevents overfitting to specific regions. CutMix combines CutOut and MixUp: remove rectangular region from image1, paste region from image2 into it, and mix labels. Creates natural-looking augmented images with mixed objects. RandAugment randomly selects augmentation operations (rotation, translation, shear, brightness, contrast) and applies them with random magnitude, ensuring diverse transformations. Example: YOLO v4 uses Mosaic extensively—the 4-image composition creates objects at different scales and positions within the image, improving small object detection by ~5% mAP. Mosaic on COCO: instead of augmenting one image per batch, mosaic creates new images from 4 COCO images, enabling 4x more effective data augmentation. CutOut alone helps but combination with Mosaic is more effective. These augmentations are particularly important for detection because they maintain object visibility while ensuring the model learns robust representations.

### Q15: Design an end-to-end object detection system for autonomous driving; discuss architecture, latency, and robustness.

**A:** Autonomous driving requires fast, accurate, robust detection with real-time constraints. Architecture: (1) Input: monocular or stereo camera (1280x720 to 1920x1080), sometimes with LiDAR/radar for 3D information. (2) Detection pipeline: YOLOv8 or EfficientDet backbone (ResNet or EfficientNet), FPN for multi-scale features, predict cars, pedestrians, cyclists, lane markings. (3) Post-processing: NMS, confidence thresholding, track detections across frames (temporal consistency). (4) Tracking: maintain object IDs across frames using Hungarian algorithm or Kalman filtering, smoothing predictions. Latency budget: <100ms total (including detection, tracking, planning) for 10 FPS decision loop, or <33ms for 30 FPS. Achievable with optimized models: quantized YOLOv8s runs ~50ms (GPU), leaving 50ms for tracking and planning. Robustness challenges: (1) Adverse weather (rain, fog, snow)—requires training on diverse conditions or domain adaptation. (2) Occlusion—cars partially hidden by other cars or trees; tracking helps recover occluded objects. (3) Night driving—infrared cameras or better backbone training on night data. (4) Small objects (distant cars)—use FPN and anchor-free methods (DINO), monitor small object mAP separately. (5) Distribution shift—model trained on highway behaves differently on city streets; use uncertainty estimation (ensemble or Bayesian) and continuous retraining on collected data. Monitoring: log detection confidence, compare to ground truth from human reviewers, retrain weekly on hard examples (confident wrong predictions). Example system: Tesla Autopilot uses 8 cameras (different views), processes frames in real-time with custom NVIDIA hardware, maintains temporal tracking, and retrains on Tesla fleet data continuously. Trade-off: accuracy vs latency—high-accuracy detector (two-stage) requires 200ms, unacceptable for driving; use one-stage detector with acceptable mAP (~40) for real-time constraint.

## Interview Cheatsheet

**Key Terms:**
- **Object Detection:** Combines classification (what?) and localization (where?); produces class labels and bounding boxes
- **Anchor-based Detection:** Predicts offsets from predefined anchor boxes; sensitive to anchor design but simple
- **Anchor-free Detection:** Directly predicts object centers and sizes; no anchor hyperparameter tuning
- **Region Proposal Network (RPN):** Learned region proposal generator in Faster R-CNN; predicts which anchors likely contain objects
- **RoI Pooling:** Crops feature map to region boundaries, applies max pooling; enables efficient per-region processing
- **Focal Loss:** Down-weights easy examples, focuses training on hard examples; addresses class imbalance in one-stage detectors
- **Non-Maximum Suppression (NMS):** Removes duplicate overlapping detections by keeping high-confidence and suppressing nearby lower-confidence boxes
- **IoU (Intersection over Union):** Measures overlap between predicted and ground-truth boxes; standard evaluation metric
- **GIoU, DIoU, CIoU:** Improved IoU variants providing gradient signal for non-overlapping or partially-overlapping boxes
- **Feature Pyramid Network (FPN):** Multi-scale feature representation combining high-resolution low-semantics with low-resolution high-semantics
- **Semantic Segmentation:** Assigns class labels to each pixel; cannot distinguish different instances
- **Instance Segmentation:** Produces masks for each distinct object; separates multiple objects of same class
- **Panoptic Segmentation:** Combines semantic and instance segmentation; full scene understanding
- **Mask R-CNN:** Extends Faster R-CNN with mask prediction branch for instance segmentation
- **RoI Align:** Differentiable improvement over RoI pooling using bilinear interpolation; improves mask quality
- **Mosaic Augmentation:** Combines 4 images into one to increase batch diversity and small object visibility

**Rapid-Fire Q&A:**
- **Q: Why is one-stage detector faster than two-stage?** **A:** One-stage predicts class/box once per region; two-stage generates proposals then processes each, adding overhead.
- **Q: What problem does focal loss solve?** **A:** Class imbalance in one-stage detectors; background examples dominate training without focal loss.
- **Q: Why use GIoU instead of IoU for loss?** **A:** IoU=0 for non-overlapping boxes (no gradient); GIoU provides signal to move box toward target.
- **Q: When should I use FPN?** **A:** Always for multi-scale detection (objects at different sizes); FPN improves accuracy by 3-5% mAP.
- **Q: Semantic vs instance segmentation?** **A:** Semantic labels pixels by class (one "dog" region); instance separates multiple dogs into distinct masks.
- **Q: What's RoI Align vs RoI Pooling?** **A:** RoI Align uses bilinear interpolation (differentiable); RoI Pooling quantizes coordinates (loses precision).
- **Q: How to optimize detector for real-time?** **A:** Use efficient backbone (MobileNet), quantize to int8, distill to smaller model, batch processing.
- **Q: Mosaic augmentation benefit?** **A:** Mixes 4 images, improves small object detection and increases effective batch diversity.
