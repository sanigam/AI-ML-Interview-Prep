# Graph Neural Networks

## Interview Anchor
- **Graph Representation and Motivation:** Why graphs need specialized architectures beyond standard neural networks; adjacency matrices, edge lists, and node features as fundamental structures
- **Message Passing Framework:** Core mechanism underlying most GNNs; propagating information through the graph structure to enable node interactions

## Key Concepts Overview
Graph Neural Networks have emerged as essential tools for structured data wherever entities and relationships matter—social networks, knowledge graphs, molecules, and recommendation systems. Interviewers assess whether you understand why standard CNNs and RNNs fail on graphs (irregular structure, variable neighbors), how message passing enables neural computation on graphs, and the evolution from spectral methods (GCN) to spatial methods (GraphSAGE, GAT). Strong candidates distinguish between node-level, edge-level, and graph-level tasks, understand efficiency challenges in GNN training on billion-node graphs, and can discuss modern innovations like graph transformers and over-smoothing problems.

---

### Q1: Explain graph representations and why standard neural networks are insufficient for graphs.

**A:** Graphs represent entities (nodes) and relationships (edges) with rich structure. Representations: adjacency matrix A (node_count x node_count, A[i,j]=1 if nodes i,j connected), edge list (list of (source, target) pairs), and node features X (node_count x feature_dim). Example: social network where nodes are users, edges are friendships, features are user attributes (age, interests). Standard neural networks (CNNs, RNNs, MLPs) fail on graphs because: (1) Variable structure—graphs have arbitrary numbers of neighbors (node degree varies), unlike images (pixels have 8 neighbors) or sequences (fixed neighborhood), (2) Permutation invariance—node ordering is arbitrary (reordering nodes shouldn't change predictions), but MLPs are order-dependent, (3) Irregular geometry—Euclidean concepts (convolution, pooling) don't directly apply; a graph is not a grid. GNNs address this by operating on neighborhoods: instead of processing each node independently (like MLP), aggregate information from neighbors (like convolution on irregular structure). Example: predicting user influence in social network requires considering followers' attributes (varying count per user), which standard neural networks cannot handle efficiently. Graph structure is essential—ignoring it and treating nodes independently loses important relational information.

### Q2: Describe the message passing framework that underlies most GNNs.

**A:** Message passing is the core mechanism: each node aggregates information from its neighbors to update its representation. The framework operates iteratively over K layers (K hops of neighborhoods): (1) At layer k, node v computes a message from each neighbor u: m_{u→v}^{(k)} = f(h_u^{(k)}, h_v^{(k)}, e_{uv}) where h_u, h_v are node representations, e_uv is edge features. (2) Aggregate messages from all neighbors: agg_v^{(k)} = AGGREGATE({m_{u→v}^{(k)} : u ∈ neighbors(v)}) using mean, sum, or max. (3) Update node representation: h_v^{(k+1)} = UPDATE(h_v^{(k)}, agg_v^{(k)}). After K layers, each node's representation incorporates K hops of neighborhood information. Aggregation and update are differentiable, enabling end-to-end learning. Advantages: naturally handles variable neighborhood sizes, permutation invariant (aggregation is order-independent), captures graph structure. Example: GCN layer h_v^{(k+1)} = σ(W * mean({h_u^{(k)} / sqrt(deg(u)*deg(v)) : u ∈ neighbors(v)} ∪ {h_v^{(k)}})) where σ is activation, W is learned weight. After 2 layers, node v's representation includes information from 2-hop neighbors. Message passing is general—GCN, GraphSAGE, GAT all follow this framework with different message functions and aggregations.

### Q3: Explain Graph Convolutional Networks (GCN) and contrast spectral vs spatial approaches.

**A:** GCN adapts convolution to graphs. Spectral approach: convolution is multiplication in Fourier domain. Compute graph Fourier basis (eigenvectors of Laplacian L = D - A where D is degree matrix), apply filters in spectral domain, transform back. Theory is elegant but computationally expensive (eigendecomposition of large graphs). Spatial approach: apply convolution directly on graph structure without spectral transform. GCN's spatial formulation: h_v^{(k+1)} = σ(W * MEAN({h_u^{(k)} : u ∈ neighbors(v)} ∪ {h_v^{(k)}})) where σ is ReLU. Simpler than spectral and scales well. GCN improves this with degree normalization: h_v^{(k+1)} = σ(W * sum_{u ∈ neighbors(v)} h_u^{(k)} / sqrt(deg(u) * deg(v))). This normalization prevents scaling issues when node degrees vary widely. Example: node classification on citation network (Cora dataset). Node features are word embeddings of paper abstracts, edges are citations. GCN 2-layer: layer 1 aggregates from direct neighbors, layer 2 aggregates from 2-hop neighbors. Output logits predict paper category. GCN achieves ~81% accuracy on Cora semi-supervised classification (only 20 labeled examples per class). Spatial approach is now standard because it's more efficient and interpretable than spectral, though both are valid. Modern variants (GAT, GraphSAGE) improve upon GCN's simple mean aggregation.

### Q4: Describe GraphSAGE: sampling, aggregation, and inductive learning.

**A:** GraphSAGE (Sample and Aggregate Graph Embeddings) addresses scalability and inductive learning (generalizing to unseen nodes). Key innovation: instead of aggregating all neighbors (expensive for nodes with thousands of neighbors), sample a fixed number of neighbors per node, aggregate those samples, enabling scalable training. Architecture: (1) Sample k neighbors for each node (e.g., k=10), (2) Recursively sample k neighbors of those k neighbors (for multiple hops), (3) Aggregate sampled neighbors using mean/LSTM/attention: agg^{(k)}_v = AGGREGATE({h_u^{(k)} : u sampled from neighbors(v)}), (4) Update: h_v^{(k+1)} = σ(W * [h_v^{(k)}, agg_v^{(k)}]). Sampling enables constant neighborhood size regardless of node degree, making training parallelizable and scalable to billion-node graphs. Inductive learning: embeddings are functions of node features and neighborhoods, not explicit node embeddings. At inference, can embed new nodes never seen during training by sampling their neighbors. Example: GraphSAGE on Amazon product network (2.3M nodes, 5M edges). Instead of propagating through all neighbors, sample 10 neighbors per node per layer. Training is efficient because computation per layer is fixed (10 aggregations), enabling 2-3 GNN layers feasibly. At inference, new products are assigned embeddings via neighbor aggregation. Comparison: GCN requires all training nodes to be present and accessible during training (transductive); GraphSAGE embeddings are generalizable (inductive). This makes GraphSAGE more practical for production systems where new nodes arrive constantly.

### Q5: Explain Graph Attention Networks (GAT) and how attention improves over fixed aggregation.

**A:** GAT replaces fixed aggregation (mean/sum) with learned attention weights. Key idea: not all neighbors are equally important—attention learns which neighbors to emphasize. For each node v, compute attention weights to each neighbor u: attention_{v→u} = softmax(LeakyReLU(W^T [h_v || h_u])) where [h_v || h_u] is concatenation, W is learned weight vector. Then aggregate with attention weights: h_v' = σ(sum_{u ∈ neighbors(v)} attention_{v→u} * W * h_u). Softmax ensures weights sum to 1. Multi-head attention (like transformers): compute K attention heads in parallel, concatenate outputs: h_v^{(k+1)} = CONCAT(head_1, ..., head_K). Advantages over GCN: adaptive—for a given node, learn which neighbors matter (e.g., for recommendation, might attend more to friends with similar preferences), interpretable (attention weights show which neighbors influence decisions). Disadvantage: more parameters (K * d^2 for K heads, d dimensions) and slightly slower than GCN (softmax computation per edge). Example: citation network where paper v is influenced by different citing papers (some highly relevant, others tangentially related). GCN averages all neighbors equally; GAT learns that highly relevant papers should have higher attention weights. On Cora, GAT achieves ~83% accuracy vs GCN's ~81% with similar parameters. Multi-head attention: K=8 heads enable the model to attend to different neighborhood patterns simultaneously (some heads focus on direct topics, others on methodology). GAT is now standard for node classification tasks requiring adaptive aggregation.

### Q6: Define node classification, link prediction, and graph classification tasks.

**A:** Node classification: predict a label for each node given graph structure and node features. Example: classify papers as ML, vision, NLP given citation network. Semi-supervised setting: only few nodes have labels. GCN/GAT propagate information from labeled nodes to unlabeled, enabling prediction with <5% labeled data. Link prediction: predict missing edges—given partial graph, predict which edges exist. Example: predict friendships in social network, or protein interactions in biology. Approaches: (1) Feature-based: extract features for node pairs (e.g., common neighbors, Jaccard similarity) and classify as edge/non-edge. (2) Embedding-based: learn node embeddings such that similar (connected) nodes have similar embeddings. Compute similarity score for each pair (e.g., dot product of embeddings), threshold to predict edges. Graph classification: predict labels for entire graphs. Example: classify molecules (graphs of atoms and bonds) as toxic/safe, or predict graph properties. Approaches: (1) Global readout: aggregate all node representations into a single graph representation (mean, sum, or max pooling of node features), then classify. (2) Hierarchical pooling: progressively pool similar nodes, reducing graph size, learning graph-level features. Example: molecule property prediction—atoms are nodes, bonds are edges, features are atom properties. Readout aggregates atom embeddings, predicts property. Task selection depends on problem: node classification for per-instance predictions, link prediction for incomplete graphs, graph classification for entire-graph properties. Difficulty scales: node classification < link prediction < graph classification (more global reasoning required).

### Q7: Explain pooling and readout operations for graph-level representations.

**A:** Node embeddings from GNNs are node-level representations; for graph-level predictions (like graph classification), aggregate nodes into a single graph representation. Basic readout: sum/mean/max all node embeddings: graph_embedding = sum/mean({h_v : v ∈ nodes}). Simple but loses node-level information—two graphs with different node embeddings but same sum cannot be distinguished. Hierarchical pooling: progressively pool similar nodes, reducing graph size while learning graph-level features. Approach: (1) Compute node scores (learned weights), (2) Select top-k scored nodes, (3) Aggregate other nodes into selected nodes (edges between non-selected and selected nodes), (4) Repeat on reduced graph. Example: DiffPool (Differentiable Pooling) learns soft cluster assignments s_v for each node v (probability of membership in cluster), progressively pooling 10% of nodes per layer. Advantages: retains hierarchical structure (graph is coarsened, not flattened), enables multiple layers of graph-level reasoning. Sophisticated readout: edge features matter. Use attention: graph_embedding = sum_{v ∈ nodes} attention_v * h_v, where attention_v is learned per-node. Multi-task readout: different downstream tasks use different readouts (node-level task keeps per-node representations, graph-level task aggregates). Example: molecule classification. Initial graph has 30 atoms (nodes). DiffPool layer 1 reduces to ~20 clusters (2/3 of nodes pooled into neighbors), layer 2 reduces to ~10 clusters. Output: graph representation that encodes hierarchical structure, enabling molecule property prediction. Without pooling, predicting molecular toxicity from 30 flattened node features is harder than from hierarchical representation.

### Q8: Describe the over-smoothing problem in deep GNNs and solutions.

**A:** Over-smoothing is a critical problem in deep GNNs: as the number of layers K increases, node embeddings converge to similar values (all nodes become indistinguishable), limiting expressivity. Mechanism: message passing aggregates neighbors' features, averaging them. Deep networks (K=10+ layers) cause exponential averaging—after 10 hops, each node's embedding is nearly the average of the entire graph (or the connected component). Mathematically, embeddings become smooth on the graph (similar nodes have similar embeddings, eventually all become identical). Over-smoothing prevents learning deep GNNs (practical limit is K=2-3 layers). Solutions: (1) Skip connections: h_v^{(k+1)} = h_v^{(k)} + UPDATE(h_v^{(k)}, agg_v^{(k)}) preserves node-specific information, preventing convergence. (2) Residual networks: similarly blend old and new representations. (3) Jumping knowledge (JK): at each node, concatenate representations from all K layers: h_v = [h_v^{(0)}, h_v^{(1)}, ..., h_v^{(K)}]. Enables the model to use both local and global information. (4) Degree normalization: use degree-aware aggregation (divide by node degree) preventing domination by high-degree neighbors. (5) Deeper GNNs with different aggregation: transformer-style self-attention instead of mean aggregation can learn to avoid over-smoothing by selectively attending to relevant neighbors. Example: 10-layer GCN without skip connections achieves poor performance (embeddings nearly identical), but with skip connections achieves competitive accuracy. JK networks enable using 7 layers by letting early layers (capturing local structure) and late layers (capturing global structure) contribute separately. Modern trend: combining skip connections + attention + normalization enables deeper GNNs (5-8 layers) while maintaining expressivity.

### Q9: Explain heterogeneous graphs and heterogeneous GNNs.

**A:** Heterogeneous graphs have multiple node types and edge types. Example: academic network with nodes {authors, papers, conferences, keywords} and edges {authors write papers, papers cite papers, papers at conferences, papers have keywords}. Standard GNNs treat all nodes and edges identically, ignoring type information. Heterogeneous GNNs propagate messages type-specifically: for each node type and edge type, apply separate learned functions. Architecture: (1) For each (source node type, edge type, target node type) triple, learn separate aggregation: h_v^{(k+1)} = UPDATE(h_v^{(k)}, sum_{(u, edge_type) : u→v} message_{edge_type}(h_u^{(k)})). (2) For node type v, aggregate messages from all incoming edge types. Example: HAN (Heterogeneous Attention Network) in academic network. When updating a paper node, separately aggregate messages from {authors who wrote it}, {papers it cites}, {conferences it was at}, {keywords}. Attention learns importance of each edge type (maybe author information matters most, keywords less). Advantages: explicitly models different relationships (citing vs authoring have different semantics), learned edge-type-specific weights reveal importance. Disadvantage: more parameters (separate weights per edge type), requires specifying node and edge types at design time. Example: company network for fraud detection. Nodes: users, accounts, transactions, merchants. Edge types: account_owner (user-account), transaction_in (account-transaction), transaction_to (transaction-merchant). Heterogeneous GNN separately processes each edge type, learning that certain user-merchant patterns indicate fraud. Compared to flattening to homogeneous graph (losing type information), heterogeneous GNN improves fraud detection by 5-10% due to edge-type-specific learning.

### Q10: Describe temporal and dynamic graphs, and challenges for GNN learning.

**A:** Temporal/dynamic graphs evolve over time: nodes or edges appear/disappear, features change. Example: social networks where friendships form/dissolve, or communication networks where messages arrive continuously. Standard GNNs assume fixed graph; temporal variants must track time. Approaches: (1) Snapshot-based: discretize time (daily snapshots), apply standard GNN to each snapshot independently or with sequential models (RNN over snapshots). Limitation: misses fine-grained temporal information if events happen within a day. (2) Event-based: represent graph as continuous stream of events (edge addition/deletion, feature update). For each event, update relevant node embeddings efficiently. (3) Temporal walk-based: sample temporal paths (walks respecting edge arrival times) instead of spatial neighborhoods. (4) Memory networks: nodes maintain memory of past interactions, updated when new events occur; useful for long-term dependency learning. Challenges: (1) Temporal scalability—cannot store all graph snapshots; efficient representations needed, (2) Concept drift—graph patterns change (friend groups shift, communication patterns evolve), (3) Node arrival—new nodes appear constantly; inductive learning is critical, (4) Feedback loops—predicting future edges affects future graph structure. Example: recommendation system as temporal graph (users, items, interactions). At time t, predict user-item edges (will user like this item?). Graph evolves as users interact. Temporal GNN learns that recent interactions matter more than old ones, and user preferences drift over time. If recommendation system is deployed, its recommendations affect future user behavior, creating feedback loop. Practical systems use sliding windows: train on last 90 days, predict next week, retrain weekly. Modern approaches (DyRep, ROLAND) maintain temporal node embeddings, updating them when new events occur, enabling efficient prediction on dynamic graphs without retraining.

### Q11: Explain knowledge graphs, graph embeddings (TransE, RotatE), and applications.

**A:** Knowledge graphs store facts as triples (subject, relation, object): e.g., (Albert Einstein, discovered, theory of relativity). Knowledge graphs are heterogeneous—different relations have different semantics (discovered ≠ born_in). Knowledge graph embedding maps entities and relations to low-dimensional vectors such that true triples have high scores. TransE: simplest approach—embed entities and relations as vectors, score triple (h, r, t) as -||h + r - t||² (translation in embedding space: h + r ≈ t). Trained with margin loss: for each true triple, push it down, for sampled false triple, push it up. RotatE: more expressive—model relation as rotation in complex vector space: score = ||h * r - t||² where * is complex multiplication. Rotation captures asymmetric relations better than translation. Example: (Obama, born_in, USA) embedded such that Obama vector rotated by born_in relation yields USA vector. Relations like "father_of" cannot be modeled by translation (adding a vector) because they're asymmetric, but can be rotation (rotating point around origin). Applications: (1) Link prediction—identify missing facts (which genes interact with protein?), (2) Entity alignment—linking entities across knowledge graphs (person in DBpedia ↔ Wikidata), (3) Question answering—represent questions as paths in knowledge graph, (4) Recommendation—knowledge graphs store user-item interactions and item metadata, GNNs learn embeddings for recommendation. Example: drug discovery. Knowledge graph contains proteins, drugs, diseases, interactions. Embedding relations like "drug_treats_disease," "protein_interacts_protein." Predict missing drug-disease edges (which drugs treat which diseases?) using TransE or RotatE. Scales to millions of entities/relations; TransE embeddings for Freebase (4.9M entities, 17K relations) computed in hours. Modern approaches (ConvKB, CompGCN) combine graph neural networks with knowledge graph embeddings for improved performance.

### Q12: Describe scalability challenges in GNN training on billion-node graphs.

**A:** Billion-node graphs (social networks, web graphs) are too large for memory; standard full-batch GNN training is infeasible. Challenges: (1) Memory: storing adjacency matrix for 1B nodes requires 10^18 bytes (impossibly large), (2) Computation: one forward pass through all nodes and edges is prohibitive, (3) Communication: distributed training requires heavy inter-node communication. Solutions: (1) Mini-batch sampling: sample subgraph from each batch instead of full graph. Sample K-hop neighborhoods for each node, train on sampled neighborhoods. GraphSAGE uses this approach—sample fixed number of neighbors per layer, enabling constant memory/computation per batch. (2) Layer-wise sampling (layer-wise sampling): for each GNN layer, independently sample neighbors, enabling arbitrary neighborhood depth without explosion. (3) Importance sampling: sample high-degree neighbors more frequently (they carry more signal), reducing variance. (4) Distributed training: partition graph (node-wise or edge-wise), place partitions on different machines, synchronize gradients. (5) GPU-accelerated frameworks: DGL, PyG support GPU-based sampling and aggregation, enabling faster training. Example: Graph classification on billion-node web graph. Standard approach: sample 512 neighbors per node per layer, train 2-layer GNN. Mini-batch size 1024 nodes requires ~512K sampled neighbors per batch (~512M edges), fits in GPU memory. Training time ~1 hour on A100 GPU. Naive full-batch would require storing entire billion-node graph adjacency (infeasible). Trade-off: sampling introduces variance (neighborhood estimation is inexact), but enables tractable training. Importance sampling reduces variance: sampling high-degree neighbors means sampled neighborhood better represents true neighborhood. Modern frameworks (GraphSAINT, PinSage) combine sampling with graph partition strategies for even better scalability.

### Q13: Explain graph transformers and their advantages over message passing GNNs.

**A:** Graph transformers apply transformer self-attention to graphs, attending over all nodes instead of just neighbors. Architecture: (1) Token embedding: encode each node as a token (node features + positional encoding), (2) Self-attention: for each node, compute attention to all other nodes: attention_{ij} = softmax(Q_i^T K_j / sqrt(d)) where Q, K are learned projections, (3) Aggregate: h_i' = sum_j attention_{ij} * V_j. Compared to message passing GNNs (aggregate only neighbors), transformers have access to global information with no message passing steps. Advantages: (1) Flexible attention—attend to relevant nodes regardless of distance, (2) No over-smoothing—each layer is independent (all-to-all attention), not iterative aggregation, (3) Expressive—transformers are more powerful than message passing (can express relationships message passing cannot). Disadvantage: quadratic complexity O(n²) in number of nodes (attention to all pairs), infeasible for very large graphs (1M+ nodes). Solutions: (1) Sparse attention—attend only to k-nearest neighbors or important nodes, reducing to O(nk), (2) Learnable attention patterns—learn which nodes to attend based on features, (3) Hierarchical attention—combine graph transformers with pooling. Example: DETR (Detection Transformer) adapted for graphs uses transformer attention on node features, learning global scene understanding without explicit message passing. Benchmark results: graph transformers slightly outperform message passing on small graphs (<10K nodes), but computational cost limits applicability. Modern trend: hybrid approaches combine message passing (efficient, local) with transformer attention (global), using few transformer layers on top of GNN layers.

### Q14: Design a recommendation system using GNNs; discuss cold-start and scalability.

**A:** Recommendation system as GNN: bipartite graph with users and items as nodes, user-item interactions as edges. Features: user attributes (demographics, history), item attributes (category, text description). GNN architecture: (1) Embed users and items using GCN or GraphSAGE, (2) Aggregate from neighbors: user embedding aggregates from items they've interacted with, item embedding aggregates from users who've interacted, (3) Score user-item pairs (e.g., dot product of embeddings), (4) Recommend top-K items. Advantages: learns user-item relationships beyond explicit interactions; if user A and B have similar interaction history, their embeddings are similar (collaborative filtering), and if items X and Y have similar features, their embeddings are similar (content-based). GNN combines both. Cold-start problem: new users/items have no interactions, cannot aggregate from neighbors. Solutions: (1) Feature-based: use content features (item category, user demographics) to initialize embeddings of new items/users (without aggregation), (2) Hybrid: combine GNN predictions (for users with history) with content-based (for cold-start users), (3) Zero-shot embeddings: pretrain item embeddings on metadata (category, description text), enabling initialization for new items. Scalability: billion-user, million-item graphs require efficient computation. Solutions: (1) GraphSAGE sampling—sample K neighbors per node, enabling fixed computation per batch, (2) Distributed training—partition users/items across machines, (3) Two-tower architecture—learn separate embeddings for users and items (no explicit interaction edges during inference), enabling independent inference, (4) Approximate nearest neighbors (ANN)—precompute all embeddings offline, at serving time use ANN index (e.g., FAISS) to find top-K similar items in milliseconds. Example: Pinterest uses GNNs for pin recommendation. Graph: 3B users, 300M pins, 300B user-pin interactions. Use GraphSAGE with neighbor sampling to train, store precomputed pin embeddings in ANN index, retrieve recommendations in <100ms. Address cold-start by using pin features (image, text) to initialize embeddings for newly uploaded pins. Feedback loop: recommended pins that users click/save are added to graph immediately (retrain weekly), enabling continuous improvement.

### Q15: Compare GNNs to transformer-based and other deep learning approaches for structured data.

**A:** GNNs vs Transformers: Transformers attend to all tokens (all-to-all attention), scale as O(n²), suit dense data with long-range dependencies (NLP, 1D sequences). GNNs attend to neighbors (sparse attention), scale as O(E) edges, suit sparse graph structure. For graphs, GNNs are more efficient; for text, transformers are more appropriate. Hybrid approaches exist: graph transformers combine both (sparse attention limited to neighborhoods). GNNs vs Graph Attention Networks (GAT): GAT is a GNN variant using learned attention instead of fixed aggregation. It's more expressive but slower. GNNs vs Convolutional Neural Networks (CNNs): CNNs assume grid structure (pixels); GNNs generalize to irregular graphs. Applying CNN to graph adjacency matrix is inefficient (adjacency is N x N but mostly sparse). GNNs leverage sparsity. GNNs vs Recurrent Neural Networks (RNNs): RNNs process sequences (1D structure); GNNs process arbitrary structure. For sequences, RNNs/Transformers are standard; for graphs, GNNs are necessary. For temporal graphs, combine GNNs (spatial structure) with RNNs (temporal dynamics). GNNs vs Kernel Methods: Kernel methods (Weisfeiler-Lehman kernels) compute graph similarity without learning; GNNs learn end-to-end via backprop, enabling better performance on downstream tasks. Kernel methods are interpretable but less flexible. Choice of method: Use GNNs for graph-structured data (molecules, social networks, knowledge graphs, recommendations). Use Transformers for sequence or dense data. Use CNNs for grid-like data (images, videos). For complex data with multiple modalities (e.g., graph + text + images), use hybrid models combining GNNs, transformers, and CNNs. Example: Multimodal GNN for e-commerce product recommendation. Product graph: items as nodes, similarity edges. Item features: category (categorical), description (text), image (visual). GNN aggregates graph neighbors, transformer encodes text, CNN encodes image, combine embeddings (concatenate or fusion network). This hybrid leverages all information sources for better recommendations.

## Interview Cheatsheet

**Key Terms:**
- **Graph:** Nodes (entities) and edges (relationships); represented as adjacency matrix, edge list, or incidence matrix
- **Message Passing:** Core GNN mechanism where nodes aggregate information from neighbors iteratively across K layers/hops
- **GCN (Graph Convolutional Network):** Applies convolution on graphs using degree-normalized aggregation; simple but effective baseline
- **Spectral vs Spatial:** Spectral GNNs use graph Fourier basis (expensive); spatial GNNs operate on graph structure directly (efficient)
- **GraphSAGE:** Scalable GNN using sampling and inductive learning; generalizes to unseen nodes
- **GAT (Graph Attention Network):** Uses learned attention weights instead of fixed aggregation; more expressive but slower
- **Node Classification:** Predict labels for nodes; semi-supervised with propagation from labeled nodes
- **Link Prediction:** Predict missing edges; embedding-based or feature-based approaches
- **Graph Classification:** Predict labels for entire graphs; requires pooling nodes to graph-level representation
- **Readout/Pooling:** Aggregate node embeddings to graph-level representation using mean/sum/max or learned pooling
- **Over-smoothing:** Problem in deep GNNs where embeddings converge to similar values; solutions include skip connections, JK networks, attention
- **Heterogeneous Graphs:** Graphs with multiple node types and edge types; require type-specific aggregation
- **Temporal/Dynamic Graphs:** Graphs that evolve over time; require efficient updates and memory networks
- **Knowledge Graph Embedding:** Map entities/relations to vectors; TransE (translation), RotatE (rotation) score triples
- **Graph Transformers:** Apply transformer self-attention to graphs; all-to-all attention but O(n²) complexity
- **Cold-start Problem:** New nodes/items with no history; use content features or zero-shot embeddings

**Rapid-Fire Q&A:**
- **Q: Why not just flatten graphs and use MLPs?** **A:** MLPs ignore structure; adjacent nodes in graph encode important relationships that flattening discards.
- **Q: Message passing aggregates which neighbors?** **A:** Immediate neighbors (1-hop); K-layer GNN aggregates K-hop neighbors.
- **Q: GCN vs GraphSAGE key difference?** **A:** GCN aggregates all neighbors (transductive); GraphSAGE samples fixed neighbors (inductive, scalable).
- **Q: Over-smoothing problem?** **A:** Deep GNNs make node embeddings similar; solutions: skip connections, JK networks, attention.
- **Q: How to score link prediction?** **A:** Embedding-based: dot product or distance of node embeddings; predict edge if score > threshold.
- **Q: Graph classification without pooling?** **A:** Loss of node-level information; pooling aggregates to graph-level representation.
- **Q: Heterogeneous graph example?** **A:** Academic network: authors (write) papers, papers (cite) papers, papers (at) conferences.
- **Q: Scalability on billion nodes?** **A:** Mini-batch sampling (GraphSAGE), importance sampling, distributed training, GPU acceleration.
